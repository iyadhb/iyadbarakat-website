Most consultants type a question into an AI tool the way they'd type it into a search box — one line, no structure — and then wonder why the answer reads like a Wikipedia summary. The fix isn't a better tool. It's a better prompt structure.

## Why one-liners fail

A one-line prompt gives the model no role, no objective, and no method. It fills the gaps with the most average pattern in its training data — which is exactly the generic output you don't want in front of a client.

## Give it a framework instead

A consulting-grade prompt does four things:

1. **Sets the role** — "You are a senior strategy consultant advising a CFO."
2. **States the objective** — what decision this output supports.
3. **Imposes a structure** — the steps you'd expect an analyst to follow.
4. **Adds constraints** — audience, length, tone, what to exclude.

That's the core idea behind the six frameworks in *The Consulting Prompt Playbook*: each one is a reusable structure for a class of consulting work, from fast daily thinking to defensible deep analysis.

## A quick example

Instead of *"How should this retailer enter the German market?"*, give the model the GOAL structure — goal, options, analysis, and a leaning recommendation — and it will pressure-test options the way a manager would, not just list them.
